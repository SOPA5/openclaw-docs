# 스킬 시스템 (Skills)

스킬은 에이전트가 어떤 일을 **더 잘하게 도와주는 전문 지식 모듈**입니다.

쉽게 말하면, 기본 에이전트가 "똑똑한 학생"이라면 스킬은 **과학 문제집, 영어 단어장, 요리 레시피 카드** 같은 것입니다.

## 📌 이 문서에서 배우는 것
- 스킬이 무엇인지
- 스킬을 어디서 찾고 설치하는지
- 내 워크스페이스 스킬과 ClawHub 스킬이 어떻게 다른지
- 스킬을 직접 만드는 가장 기본 구조

:::tip 쉽게 이해하기
스킬은 앱을 새로 설치하는 느낌과 비슷합니다. 에이전트 몸체는 그대로 두고, **특정 일을 더 잘하는 능력만 추가**하는 방식입니다.
:::

## 스킬이 왜 필요할까?

에이전트는 원래도 많은 일을 할 수 있습니다. 하지만 어떤 일은 **정해진 순서, 전문 용어, 자주 쓰는 도구**가 있어서 따로 설명해 주면 훨씬 잘합니다.

예를 들면:
- 날씨를 자주 묻는다면 → `weather` 스킬
- GitHub 이슈나 PR을 자주 다룬다면 → `github` 스킬
- 유튜브를 정리해서 메모로 남기고 싶다면 → `youtube-to-obsidian` 같은 스킬

즉, 스킬은 에이전트에게 **작업 설명서**를 붙여 주는 것입니다.

## ClawHub 레지스트리

ClawHub는 스킬을 모아둔 **스킬 상점 같은 레지스트리**입니다.

- 주소: `clawhub.ai`
- 하는 일: 공개된 스킬을 검색하고 설치할 수 있게 도와줌
- 느낌: npm 레지스트리나 앱 스토어처럼, 필요한 기능을 찾아서 가져오는 곳

:::tip
필요한 스킬이 이미 ClawHub에 있으면, 처음부터 직접 만들지 않아도 됩니다. 먼저 있는지 찾아보는 게 빠릅니다.
:::

## 스킬 설치하기

ClawHub에서 스킬을 설치할 때는 보통 이렇게 씁니다.

```bash
clawhub install <slug>
```

예시:

```bash
clawhub install weather
```

여기서 `<slug>`는 스킬의 짧은 이름입니다.

## 설치된 스킬 목록 보기

지금 사용할 수 있는 스킬 목록은 이렇게 확인합니다.

```bash
openclaw skills list
```

이 명령은 보통 다음 같은 질문에 답해 줍니다.
- 어떤 스킬이 이미 깔려 있지?
- 로컬(workspace) 스킬도 보이나?
- 상태가 정상인가?

## 스킬의 기본 구조

직접 스킬을 만들 때 가장 중요한 파일은 보통 `SKILL.md`입니다.

그리고 이 파일 맨 위에는 **YAML frontmatter**가 있어야 합니다.

예시:

```md
---
name: my-skill
description: Do one job really well.
---

# My Skill

이 스킬은 특정 작업을 잘하도록 도와줍니다.
```

### 왜 frontmatter가 중요할까?

frontmatter는 스킬의 **이름표**입니다.

여기에 보통 이런 정보가 들어갑니다.
- `name`: 스킬 이름
- `description`: 어떤 일을 잘하는지 설명

OpenClaw는 이 정보를 보고 스킬을 식별하고, 언제 쓰면 좋은지 판단합니다.

## workspace skills vs ClawHub skills

OpenClaw에는 크게 두 종류의 스킬이 있습니다.

### 1) workspace skills
내 컴퓨터의 워크스페이스 안에 직접 두는 스킬입니다.

- 팀이나 개인 환경에 맞게 자유롭게 수정 가능
- 실험용, 내부용, 나만의 자동화에 좋음
- 보통 로컬 규칙이나 운영 문서를 담기 좋음

### 2) ClawHub skills
ClawHub에서 설치하는 공개 스킬입니다.

- 이미 만들어진 걸 바로 가져다 쓰기 좋음
- 다른 사람도 같은 스킬을 쉽게 재사용 가능
- 업데이트와 배포 흐름이 더 분명함

:::tip 한 줄 정리
- **workspace skills** = 우리 집 서랍에 넣어둔 맞춤 도구
- **ClawHub skills** = 가게에서 사 온 공용 도구
:::

## 실전 예시

### `weather`
날씨 질문이 들어오면 지역, 현재 날씨, 예보를 더 잘 다루도록 도와줍니다.

### `github`
이슈, PR, CI, 리뷰 확인처럼 GitHub 관련 작업을 더 정확한 흐름으로 처리하게 도와줍니다.

### `youtube-watcher` 또는 유사한 유튜브 계열 스킬
유튜브 링크를 정리하거나, 새 영상 확인, 메모 저장 같은 일을 더 쉽게 해 줍니다.

## 스킬을 쓸 때 좋은 습관

1. 먼저 기본 기능으로 되는지 본다
2. 반복되는 작업이면 스킬을 찾는다
3. 공개 스킬이 없으면 workspace skill을 만든다
4. 스킬 설명은 짧고 또렷하게 쓴다

## ✅ 성공 확인
- 스킬이 "에이전트를 더 잘하게 만드는 모듈"이라는 뜻을 설명할 수 있습니다.
- `clawhub install <slug>`와 `openclaw skills list`의 역할을 구분할 수 있습니다.
- `SKILL.md`와 YAML frontmatter가 왜 필요한지 이해했습니다.
- workspace skill과 ClawHub skill의 차이를 말할 수 있습니다.
