# 컨텍스트 관리 (Context)

컨텍스트는 모델이 **지금 이 순간 참고하는 재료 묶음**입니다.

## 📌 이 문서에서 배우는 것
- 컨텍스트에 들어오는 재료
- 왜 compaction과 pruning이 필요한지
- 길어진 작업에서 품질을 지키는 방법

:::tip 쉽게 이해하기
컨텍스트는 **책상 위에 올려둔 자료 더미**와 같습니다. 너무 많으면 어수선하고, 너무 적으면 필요한 정보가 부족합니다. 적당히 잘 고르는 게 중요합니다.
:::

## 핵심 이해
- 최근 대화, 시스템 지침, 메모리, 도구 결과가 함께 들어옵니다.
- 너무 큰 출력은 축약하거나 정리해서 넣어야 합니다.
- 중요한 사실은 남기고, 덜 중요한 흔적은 줄여야 품질과 비용이 함께 좋아집니다.

## 실전에서 왜 중요한가
- 구조를 이해하면 설정 실수를 빨리 찾을 수 있습니다.
- 문제를 봤을 때 “모델 문제인지, 세션 문제인지, 도구 문제인지” 더 쉽게 구분할 수 있습니다.
- 여러 문서를 읽을 때 머릿속 지도가 생깁니다.

## 짧은 예시
- **예시 1**: 긴 보고서 작업을 할 때는 세션이 흐름을 이어주고, 컨텍스트는 지금 필요한 부분만 모델에게 보여줍니다.
- **예시 2**: 다른 작업자에게 일을 나누면 에이전트와 세션 구조를 함께 이해해야 헷갈리지 않습니다.
