# 세션 관리 (Sessions)

세션은 OpenClaw가 **어떤 대화를 어디까지 이어서 기억할지** 정하는 상자입니다.

## 📌 이 문서에서 배우는 것
- 메인 세션과 격리 세션 차이
- 세션 안에 무엇이 들어가는지
- 왜 pruning과 정리가 필요한지

:::tip 쉽게 이해하기
세션은 **공책 한 권**과 비슷합니다. 같은 공책 안에서는 앞에 적은 내용이 이어지고, 다른 공책으로 바꾸면 주제를 깔끔하게 분리할 수 있습니다.
:::

## 핵심 이해
- 개인 DM은 메인 세션으로 이어지는 경우가 많습니다.
- 그룹 대화는 개인 문맥과 섞이지 않게 격리하는 편이 안전합니다.
- 너무 길어진 세션은 pruning과 compaction으로 가볍게 관리합니다.

## 실전에서 왜 중요한가
- 구조를 이해하면 설정 실수를 빨리 찾을 수 있습니다.
- 문제를 봤을 때 “모델 문제인지, 세션 문제인지, 도구 문제인지” 더 쉽게 구분할 수 있습니다.
- 여러 문서를 읽을 때 머릿속 지도가 생깁니다.

## 짧은 예시
- **예시 1**: 긴 보고서 작업을 할 때는 세션이 흐름을 이어주고, 컨텍스트는 지금 필요한 부분만 모델에게 보여줍니다.
- **예시 2**: 다른 작업자에게 일을 나누면 에이전트와 세션 구조를 함께 이해해야 헷갈리지 않습니다.
