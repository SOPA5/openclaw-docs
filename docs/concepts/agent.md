# 에이전트 런타임 (Agent Runtime)

에이전트는 단순히 말만 하는 챗봇이 아니라, **도구를 쓰고 일을 진행하는 작업자**입니다.

## 📌 이 문서에서 배우는 것
- 사용자 요청을 해석하는 흐름
- 도구를 쓰는 루프
- 다른 세션이나 작업자에게 위임하는 방법

:::tip 쉽게 이해하기
에이전트는 **팀원 한 명**처럼 생각하면 쉽습니다. 질문을 듣고, 필요한 자료를 찾고, 다른 사람에게 일을 부탁하고, 마지막에 결과를 정리해 주는 역할입니다.
:::

## 핵심 이해
- 사용자 요청 해석 → 컨텍스트 확인 → 도구 호출 → 결과 해석 → 다음 행동 결정
- 필요하면 `sessions_spawn`으로 다른 작업자를 띄웁니다.
- 길고 복잡한 작업일수록 오케스트레이션 능력이 중요합니다.

## 실전에서 왜 중요한가
- 구조를 이해하면 설정 실수를 빨리 찾을 수 있습니다.
- 문제를 봤을 때 “모델 문제인지, 세션 문제인지, 도구 문제인지” 더 쉽게 구분할 수 있습니다.
- 여러 문서를 읽을 때 머릿속 지도가 생깁니다.

## 짧은 예시
- **예시 1**: 긴 보고서 작업을 할 때는 세션이 흐름을 이어주고, 컨텍스트는 지금 필요한 부분만 모델에게 보여줍니다.
- **예시 2**: 다른 작업자에게 일을 나누면 에이전트와 세션 구조를 함께 이해해야 헷갈리지 않습니다.
