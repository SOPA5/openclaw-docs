# 기억 시스템 (Memory)

메모리는 긴 대화가 끝나도 **계속 남겨둘 중요한 사실**을 보관하는 층입니다.

## 📌 이 문서에서 배우는 것
- 어떤 정보가 메모리로 올라가는지
- 파일형 기억과 구조화 기억의 차이
- 세션과 메모리의 역할 분담

:::tip 쉽게 이해하기
세션이 오늘 쓰는 **칠판**이라면, 메모리는 오래 두고 보는 **노트**에 가깝습니다. 매번 다시 설명할 필요가 없는 사실을 따로 적어두는 셈입니다.
:::

## 핵심 이해
- 자주 다시 써야 하는 사실은 메모리로 올리는 편이 좋습니다.
- 오늘 잠깐 필요했던 잡다한 내용은 세션에만 남아도 됩니다.
- 메모리를 너무 많이 넣으면 오히려 잡음이 될 수 있어 핵심만 남기는 게 좋습니다.

## 실전에서 왜 중요한가
- 구조를 이해하면 설정 실수를 빨리 찾을 수 있습니다.
- 문제를 봤을 때 “모델 문제인지, 세션 문제인지, 도구 문제인지” 더 쉽게 구분할 수 있습니다.
- 여러 문서를 읽을 때 머릿속 지도가 생깁니다.

## 짧은 예시
- **예시 1**: 긴 보고서 작업을 할 때는 세션이 흐름을 이어주고, 컨텍스트는 지금 필요한 부분만 모델에게 보여줍니다.
- **예시 2**: 다른 작업자에게 일을 나누면 에이전트와 세션 구조를 함께 이해해야 헷갈리지 않습니다.
