# 아키텍처 개요 (Architecture)

아키텍처는 OpenClaw가 **채널, 모델, 도구, 세션을 어떻게 연결해 돌아가는지** 보여주는 큰 그림입니다.

## 📌 이 문서에서 배우는 것
- OpenClaw를 이루는 핵심 블록
- 사용자 요청이 어디를 거쳐 가는지
- 왜 Gateway가 중심에 있는지

:::tip 쉽게 이해하기
아키텍처는 **도시 지도**와 비슷합니다. 건물 하나만 보면 길을 잃기 쉽지만, 지도를 보면 어디가 입구이고 어디가 중심인지 한눈에 보입니다.
:::

## 핵심 이해
- 사용자는 채널에서 말을 걸고, Gateway가 요청을 받아 처리 흐름을 묶습니다.
- Provider는 두뇌를, Tool은 손발을, Session은 기억 흐름을 맡습니다.
- 구조를 이해하면 어디서 문제가 났는지 찾기가 쉬워집니다.

## 실전에서 왜 중요한가
- 구조를 이해하면 설정 실수를 빨리 찾을 수 있습니다.
- 문제를 봤을 때 “모델 문제인지, 세션 문제인지, 도구 문제인지” 더 쉽게 구분할 수 있습니다.
- 여러 문서를 읽을 때 머릿속 지도가 생깁니다.

## 짧은 예시
- **예시 1**: 긴 보고서 작업을 할 때는 세션이 흐름을 이어주고, 컨텍스트는 지금 필요한 부분만 모델에게 보여줍니다.
- **예시 2**: 다른 작업자에게 일을 나누면 에이전트와 세션 구조를 함께 이해해야 헷갈리지 않습니다.
