# 시스템 프롬프트 (System Prompt)

시스템 프롬프트는 OpenClaw가 **어떤 성격과 규칙으로 행동할지** 정하는 맨 위 지시문입니다.

## 📌 이 문서에서 배우는 것
- 무슨 정보가 합쳐지는지
- 역할·안전·도구 지침이 어떻게 작동하는지
- 왜 사용자 요청보다 더 위에 있는 규칙이 필요한지

:::tip 쉽게 이해하기
시스템 프롬프트는 **게임의 룰북**과 비슷합니다. 플레이어가 똑같아도 규칙이 다르면 행동이 달라지듯, 모델도 시스템 프롬프트에 따라 말투와 우선순위가 달라집니다.
:::

## 핵심 이해
- 기본 정체성, 안전 규칙, 도구 사용 지침이 함께 합쳐집니다.
- 사용자 요청과 충돌할 때는 더 높은 우선순위 규칙이 먼저 적용됩니다.
- 좋은 시스템 프롬프트는 장황한 소개보다 행동 기준을 분명히 줍니다.

## 실전에서 왜 중요한가
- 구조를 이해하면 설정 실수를 빨리 찾을 수 있습니다.
- 문제를 봤을 때 “모델 문제인지, 세션 문제인지, 도구 문제인지” 더 쉽게 구분할 수 있습니다.
- 여러 문서를 읽을 때 머릿속 지도가 생깁니다.

## 짧은 예시
- **예시 1**: 긴 보고서 작업을 할 때는 세션이 흐름을 이어주고, 컨텍스트는 지금 필요한 부분만 모델에게 보여줍니다.
- **예시 2**: 다른 작업자에게 일을 나누면 에이전트와 세션 구조를 함께 이해해야 헷갈리지 않습니다.
