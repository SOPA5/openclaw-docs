# Nodes (모바일/원격 노드)

노드는 OpenClaw에 연결된 **다른 기기**입니다.

예를 들면:
- iPhone
- Android 폰
- 다른 컴퓨터
- 화면 없이 돌아가는 작은 원격 머신

즉, 내 메인 OpenClaw가 한곳에 있고,
다른 기기들이 거기에 연결되어 **눈, 귀, 손**처럼 도와주는 구조예요.

## 📌 이 문서에서 배우는 것
- 노드가 무엇인지
- iOS와 Android를 node로 연결하는 기본 흐름
- 노드가 해줄 수 있는 일
- Talk Mode와 노드의 관계
- QR 코드 페어링이 무엇인지
- `openclaw devices list`, `openclaw devices approve` 를 언제 쓰는지
- 밖에서 휴대폰으로 AI 비서를 쓰는 실제 예시

:::tip 쉽게 이해하기
- **Gateway** = 본부
- **Node** = 현장 요원

본부가 생각하고,
현장 요원이 카메라·마이크·위치 같은 현실 세계 정보를 가져오는 느낌이라고 생각하면 쉬워요.
:::

## 노드란?

노드는 Gateway에 연결되는 **보조 기기**입니다.

중요한 점은 이것입니다.

- **Gateway**는 중심 통로예요.
- **Node**는 기능을 가져오는 연결 기기예요.

그래서 휴대폰은 보통 OpenClaw 전체를 혼자 운영하는 본체가 아니라,
이미 켜져 있는 OpenClaw에 붙어서 기능을 확장하는 역할을 합니다.

예를 들면 휴대폰이 이렇게 도와줄 수 있어요.
- 카메라로 지금 보는 장면 보내기
- 마이크로 음성을 전달하기
- 위치 정보를 알려주기
- 알림을 띄우기

## 노드로 연결하면 좋은 이유

데스크톱만 있으면 책상 앞에서만 쓰기 쉽습니다.
하지만 노드를 붙이면 OpenClaw가 **현실 세계와 더 잘 연결**됩니다.

예를 들면:
- 밖에서 휴대폰 카메라로 물건을 보여주기
- 이동 중에 음성으로 질문하기
- 다른 컴퓨터에서 명령 실행하기
- 휴대폰 알림으로 결과 받기

## iOS 노드 연결

iPhone이나 iPad는 OpenClaw iOS 앱을 통해 node로 연결할 수 있습니다.

### 보통 준비물
- OpenClaw Gateway가 켜져 있는 컴퓨터 또는 서버
- iPhone/iPad
- 같은 네트워크이거나, 안전한 원격 연결 경로

### 기본 흐름
1. Gateway를 켭니다.
2. iOS 앱에서 Gateway를 찾거나 주소를 입력합니다.
3. 필요하면 QR 코드나 설정 코드를 사용합니다.
4. Gateway 쪽에서 페어링 요청을 승인합니다.
5. 연결 후 카메라/음성/화면 기능을 사용할 수 있습니다.

### iOS에서 기대할 수 있는 기능
- 카메라 사용
- Talk Mode 음성 대화
- 화면 관련 흐름
- Canvas 기반 UI 표시

:::tip iOS는 이렇게 생각하면 쉬워요
아이폰이 OpenClaw의 **작은 리모컨 + 카메라 + 마이크**가 된다고 생각하면 됩니다.
:::

## Android 노드 연결

Android도 비슷한 방식으로 node로 연결할 수 있습니다.

### 기본 흐름
1. Gateway를 준비합니다.
2. Android 앱에서 Gateway에 연결합니다.
3. 설정 코드 또는 직접 주소 입력을 사용합니다.
4. Gateway에서 승인합니다.
5. 연결 후 카메라, 음성, 기기 기능을 활용합니다.

### Android에서 기대할 수 있는 기능
- 카메라 사용
- 음성 입력/출력
- Canvas 기반 화면
- 일부 기기 명령 및 확장 기능

:::tip Android는 기기별 차이가 있을 수 있어요
Android는 제조사와 버전이 다양해서,
권한 화면이나 세부 메뉴가 조금씩 다를 수 있습니다.
그래도 큰 흐름은 **연결 요청 → Gateway 승인 → 기능 허용** 순서로 같습니다.
:::

## 노드가 해줄 수 있는 대표 기능

| 기능 | 뜻 |
|---|---|
| 카메라 | 휴대폰 카메라로 장면을 찍어 에이전트에 전달 |
| 오디오 | 마이크 음성을 보내고 응답을 듣기 |
| 위치 | 지금 있는 장소 정보를 기반으로 도움 받기 |
| 알림 | 결과나 안내를 기기로 보내기 |
| Canvas | 노드 화면에 간단한 UI를 표시 |

이 말은 곧,
OpenClaw가 글만 주고받는 비서가 아니라 **현실 세계와 연결된 비서**가 될 수 있다는 뜻입니다.

## Talk Mode와 노드

Talk Mode는 말 그대로 **말로 대화하는 모드**예요.

노드가 붙어 있으면,
특히 휴대폰에서 Talk Mode를 훨씬 자연스럽게 쓸 수 있습니다.

예를 들면:
- 길을 걷다가 질문하기
- 손이 바쁠 때 음성으로 요청하기
- 화면을 오래 안 보고도 대화하기

즉,
**노드 + Talk Mode** 조합은 OpenClaw를 더 "사람처럼 바로 부를 수 있는 비서"에 가깝게 만들어 줍니다.

:::tip 이런 상황에서 특히 좋아요
요리 중, 운전 전후, 이동 중, 짐을 들고 있을 때처럼
손으로 타이핑하기 불편한 순간에 Talk Mode가 정말 편합니다.
:::

## QR 코드 페어링이란?

QR 코드 페어링은 휴대폰이 Gateway를 쉽게 찾고 연결하도록 돕는 방법입니다.

긴 주소를 손으로 하나하나 입력하는 대신,
**코드를 찍어서 빠르게 연결 정보**를 가져오는 거예요.

필요할 때는 이런 명령을 사용할 수 있습니다.

```bash
openclaw qr
```

설정 코드만 보고 싶다면:

```bash
openclaw qr --setup-code-only
```

이 흐름은 특히 iOS 같은 모바일 연결에서 편리합니다.

## `openclaw devices list`, `openclaw devices approve`

노드가 연결 요청을 보내면,
Gateway는 그 요청을 바로 무조건 믿지 않고 **승인 대기** 상태로 둘 수 있습니다.

그걸 확인하고 승인할 때 자주 쓰는 명령이 이 두 개예요.

### 1) 요청 보기

```bash
openclaw devices list
```

이 명령은:
- 대기 중인 페어링 요청
- 이미 연결된 기기 목록

을 보여줍니다.

### 2) 요청 승인하기

```bash
openclaw devices approve <requestId>
```

또는 가장 최근 요청을 승인할 수도 있습니다.

```bash
openclaw devices approve --latest
```

### 왜 이 단계가 중요할까?
내 Gateway에 아무 기기나 함부로 붙으면 위험할 수 있기 때문이에요.

그래서 "이 기기를 정말 연결해도 되나?"를 확인하는 문지기 역할을 합니다.

:::tip 승인 전에 다시 확인하세요
기기가 다시 요청을 보내면 `requestId`가 바뀔 수 있어요.
그래서 보통 **승인 직전에 `openclaw devices list`를 한 번 더 보는 습관**이 좋습니다.
:::

## 기본 연결 흐름 한 번에 보기

```text
휴대폰 앱에서 연결 요청
        ↓
Gateway가 페어링 요청을 받음
        ↓
openclaw devices list 로 요청 확인
        ↓
openclaw devices approve <requestId> 로 승인
        ↓
노드 연결 완료
```

## 실전 예시: 밖에서 스마트폰으로 AI 비서에게 질문하기

상황:
집이나 사무실 컴퓨터에 OpenClaw Gateway가 이미 켜져 있습니다.
나는 밖에 있고, 손에는 스마트폰만 있습니다.

### 하고 싶은 일
- 오늘 미팅 장소 근처 카페 찾기
- 지금 찍은 사진 보고 설명 듣기
- 음성으로 메모 정리 부탁하기

### 실제 흐름
1. 스마트폰이 node로 Gateway에 연결되어 있습니다.
2. 나는 Talk Mode로 말합니다.
   - "지금 근처 조용한 카페 찾아줘"
3. 필요하면 위치 정보가 함께 활용됩니다.
4. 사진이 필요하면 바로 카메라로 보여줄 수 있습니다.
5. 결과는 음성 또는 메시지로 다시 받습니다.

이렇게 되면 OpenClaw는 책상 위 챗봇이 아니라,
**밖에서도 따라다니는 AI 비서**처럼 느껴집니다.

## 자주 헷갈리는 포인트

### 1) 노드가 Gateway인가요?
아니요.
노드는 Gateway에 붙는 **연결 기기**입니다.

### 2) 모바일 앱만 있으면 다 되나요?
보통은 먼저 연결할 **Gateway 본체**가 있어야 합니다.

### 3) 노드가 여러 개일 수도 있나요?
네.
휴대폰도, 다른 컴퓨터도, 별도 노드 호스트도 연결될 수 있습니다.

### 4) 노드는 왜 승인 과정이 있나요?
보안 때문입니다.
허락된 기기만 연결되게 하려는 거예요.

## ✅ 성공 확인
- 노드가 "OpenClaw에 연결된 보조 기기"라는 뜻을 설명할 수 있습니다.
- Gateway와 node의 역할 차이를 구분할 수 있습니다.
- iOS/Android 연결 흐름을 큰 그림으로 이해했습니다.
- `openclaw devices list` 와 `openclaw devices approve` 가 왜 필요한지 설명할 수 있습니다.
- Talk Mode와 모바일 node를 함께 쓰는 장점을 이해했습니다.
