# 설정 마법사

처음 OpenClaw를 켤 때 가장 쉬운 시작 방법은 **설정 마법사**를 쓰는 것입니다.

복잡한 설정 파일을 처음부터 손으로 만들 필요는 없습니다. 마법사가 질문을 하나씩 보여주고, 여러분은 **고르기만** 하면 됩니다.

:::tip 쉬운 비유
설정 마법사는 새 휴대폰을 처음 켤 때 나오는 **처음 설정 도우미**와 비슷해요.
언어를 고르고, Wi‑Fi를 연결하고, 계정을 넣듯이 OpenClaw도 필요한 것들을 차례대로 묻습니다.
:::

## 이 문서에서 하는 일

- 마법사를 어떻게 실행하는지 보기
- 마법사가 무엇을 설정해주는지 알기
- 다시 바꾸고 싶을 때 어디를 열면 되는지 알기

## 먼저 알아두기

OpenClaw를 아직 설치하지 않았다면, 설치 명령과 플랫폼별 준비는 이 문서에서 길게 반복하지 않을게요.
자세한 내용은 [설치 개요](/install/)를 먼저 보면 됩니다.

## 1) 마법사 실행하기

터미널을 열고 아래 명령을 실행하세요.

```bash
openclaw onboard --install-daemon
```

### 이 명령이 하는 일

- `openclaw onboard`: 처음 설정을 시작합니다.
- `--install-daemon`: OpenClaw가 백그라운드에서 자동으로 실행되도록 등록합니다.

운영체제에 따라 자동 등록 방식이 다릅니다.

- macOS: LaunchAgent
- Linux: systemd
- Windows: 환경에 따라 권장 방식이 다름

## 2) 마법사가 물어보는 것

마법사는 보통 아래 순서로 진행됩니다.

| 단계 | 쉬운 설명 |
|---|---|
| 모델 / 로그인 | 어떤 AI를 쓸지, 어떻게 로그인할지 고릅니다. |
| 워크스페이스 | 에이전트가 파일을 읽고 작업할 기본 폴더를 정합니다. |
| 게이트웨이 | OpenClaw의 중심 통로를 켤 방법을 정합니다. |
| 채널 연결 | Telegram, Discord 같은 대화 창을 연결합니다. |
| 서비스 등록 | 컴퓨터를 켤 때 같이 시작되게 할지 정합니다. |
| 추천 설정 | 처음 쓰기 좋은 기본값을 넣어줍니다. |

## 3) 진행할 때 이렇게 생각하면 쉬워요

### 모델 / 로그인
여기서는 “어느 두뇌를 쓸까?”를 고른다고 생각하면 됩니다.

처음에는 **권장값**을 고르면 충분합니다.
나중에 더 빠른 모델, 더 강한 모델로 바꾸는 것도 가능합니다.

### 워크스페이스
워크스페이스는 OpenClaw의 **작업 책상**입니다.

보통 기본 경로를 그대로 써도 괜찮습니다.
예를 들어 아래처럼 쓰는 경우가 많습니다.

- `~/.openclaw/workspace`

### 게이트웨이
게이트웨이는 OpenClaw의 **중앙 통로**입니다.
메시지, 도구, 모델을 서로 이어줍니다.

### 채널 연결
이 단계에서 여러분은 “어디서 OpenClaw와 대화할지”를 정합니다.

예를 들면:
- Telegram에서 대화하기
- Discord에서 대화하기
- Slack에서 대화하기

처음엔 **자주 쓰는 채널 하나만** 연결해도 충분합니다.

## 4) 중간에 멈춰도 괜찮아요

설정을 하다 보면 이런 일이 있을 수 있어요.

- 로그인 방식이 헷갈림
- 토큰이나 키를 어디서 가져오는지 모름
- 어떤 채널을 먼저 연결해야 할지 고민됨

이럴 때는 전부 한 번에 끝내려고 하지 않아도 됩니다.
우선 기본 설정만 마친 뒤, 나중에 하나씩 다시 열어 바꿔도 됩니다.

## 5) 나중에 다시 바꾸기

마법사를 처음부터 다시 돌리지 않아도 됩니다.
필요한 부분만 고쳐도 됩니다.

```bash
# 설정 다시 열기
openclaw configure

# 게이트웨이 서비스 다시 설치/정비
openclaw gateway install

# 문제 진단
openclaw doctor
```

## 6) 끝났는지 확인하기

설정이 끝났다면 아래 명령으로 상태를 확인하세요.

```bash
openclaw gateway status
openclaw dashboard
```

:::tip ✅ 성공하면 이렇게 보입니다
- `openclaw gateway status`를 쳤을 때 `running` 같은 정상 상태가 보입니다.
- 대시보드가 열리거나, 연결된 채널에서 첫 메시지를 보낼 준비가 됩니다.
- “이제 뭘 하지?”보다 “이제 바로 써볼 수 있겠다”라는 느낌이 들면 잘 된 거예요.
:::

## 자주 막히는 지점

### 서비스가 안 켜져요
`openclaw doctor`를 먼저 실행해 보세요.
대부분은 누락된 설정이나 권한 문제를 알려줍니다.

### 채널 연결이 헷갈려요
채널은 나중에 다시 연결할 수 있습니다.
먼저 마법사를 마치고, 그다음 [채널 페어링](/start/pairing) 문서로 넘어가면 됩니다.

### 플랫폼마다 화면이 달라요
괜찮습니다. 큰 흐름은 같고, 버튼 이름이나 서비스 등록 방식만 조금 다를 수 있습니다.
자세한 차이는 [플랫폼 개요](/platforms/)와 각 플랫폼 문서를 보면 됩니다.
