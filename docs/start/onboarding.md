# 온보딩 (macOS 앱)

이 문서는 **macOS 앱으로 OpenClaw를 처음 시작하는 사람**을 위한 안내서입니다.

터미널이 익숙하지 않아도 괜찮습니다.
앱이 보여주는 순서대로 따라가면 됩니다.

> CLI로 시작하고 싶다면 [설정 마법사](/start/wizard)를 보면 됩니다.

:::tip 쉬운 비유
macOS 앱 온보딩은 새 집에 들어가기 전에 **현관 비밀번호를 정하고, 전등 스위치 위치를 익히고, 열쇠를 맞춰보는 과정**과 비슷해요.
처음 한 번만 잘 해두면 그다음부터는 훨씬 편해집니다.
:::

## 이 문서에서 하는 일

- macOS 앱을 열고 처음 설정하기
- 보안 경고와 권한 요청 이해하기
- 로컬 모드와 원격 모드 중 무엇을 고를지 정하기

## 먼저 알아두기

앱 설치 방법과 다운로드 경로는 여기서 길게 반복하지 않을게요.
자세한 내용은 [설치 개요](/install/)를 참고하세요.

## 1) 앱을 처음 열면 생기는 일

처음 실행하면 macOS가 “이 앱을 실행해도 괜찮은지”를 물을 수 있습니다.
이건 이상한 일이 아니라, Mac이 여러분을 보호하려고 확인하는 과정입니다.

### 보안 경고가 보이면

- `시스템 설정 > 개인정보 보호 및 보안`에서 실행을 허용하세요.
- 또는 상황에 따라 아래 명령으로 격리 속성을 제거할 수 있습니다.

```bash
xattr -dr com.apple.quarantine /Applications/OpenClaw.app
```

> 이 명령이 낯설다면 무조건 먼저 쓰기보다, macOS 화면 안내를 보고 허용하는 방식부터 시도하는 것이 더 쉽습니다.

## 2) 로컬 모드 vs 원격 모드 고르기

앱은 보통 두 가지 길을 보여줍니다.

### 로컬 모드
**지금 쓰고 있는 이 Mac 안에서** OpenClaw의 중심 기능을 실행합니다.

이런 분께 잘 맞아요.
- 개인용으로 빠르게 시작하고 싶은 사람
- 서버 없이 내 Mac만으로 써보고 싶은 사람
- 설정을 단순하게 유지하고 싶은 사람

### 원격 모드
다른 컴퓨터나 서버에서 돌아가는 OpenClaw에 **이 앱이 연결만** 합니다.

이런 분께 잘 맞아요.
- 집 서버나 클라우드 서버를 이미 쓰는 사람
- 여러 기기에서 같은 환경을 연결하고 싶은 사람
- 항상 켜져 있는 머신이 따로 있는 사람

**처음이라면 보통 로컬 모드가 더 쉽습니다.**

## 3) 권한 요청 허용하기

OpenClaw가 Mac을 도와 움직이려면 몇 가지 권한이 필요합니다.
각 권한은 이유가 있습니다.

| 권한 | 왜 필요한가요? |
|---|---|
| 자동화 (Automation) | 다른 앱을 열거나 제어하기 위해 필요합니다. |
| 손쉬운 사용 (Accessibility) | 클릭, 입력 같은 보조 조작에 필요합니다. |
| 화면 기록 (Screen Recording) | 화면 상태를 보고 도와주기 위해 필요합니다. |
| 마이크 (Microphone) | 음성 입력이나 음성 기능을 쓸 때 필요합니다. |
| 알림 (Notifications) | 작업이 끝났을 때 알려주기 위해 필요합니다. |

### 권한 허용 팁

처음에는 권한이 많아 보여서 겁날 수 있습니다.
하지만 OpenClaw가 하려는 일과 연결해서 보면 이해가 쉬워요.

- 화면을 봐야 도와줄 수 있고
- 클릭하려면 조작 권한이 필요하고
- 끝났다고 알려주려면 알림 권한이 필요합니다

## 4) CLI도 같이 설치할까요?

macOS 앱만 써도 시작할 수 있지만, 나중에 조금 더 깊게 쓰고 싶다면 CLI도 있으면 편합니다.

```bash
npm install -g openclaw@latest
```

설치 후에는 아래처럼 확인해 보세요.

```bash
openclaw --version
openclaw doctor
openclaw gateway status
```

:::tip ✅ 성공하면 이렇게 보입니다
- 앱이 정상적으로 열리고, 초기 설정 화면이 끝까지 진행됩니다.
- `openclaw --version`에서 버전 번호가 보입니다.
- `openclaw doctor`에서 큰 문제 없이 통과 메시지가 보입니다.
- `openclaw gateway status`에서 `running` 같은 정상 상태가 보입니다.
:::

## 5) 온보딩이 끝나면 무엇을 하게 되나요?

설정이 끝나면 보통 이런 상태가 됩니다.

- OpenClaw가 쓸 기본 통로가 준비됨
- 필요한 권한이 연결됨
- 대화를 시작할 채널이나 앱 흐름이 준비됨

쉽게 말해, **이제 진짜로 써볼 준비가 된 상태**입니다.

## 6) 나중에 설정을 바꾸고 싶다면

처음 선택이 영원히 고정되는 것은 아닙니다.
언제든 다시 바꿀 수 있습니다.

```bash
openclaw configure
```

예를 들어 이런 걸 바꿀 수 있어요.

- 연결할 채널
- 모델 / 로그인 방식
- 게이트웨이 관련 설정

## 자주 헷갈리는 질문

### 꼭 CLI를 설치해야 하나요?
아니요. 앱 중심으로만 시작해도 됩니다.
다만 문제를 점검하거나 고급 기능을 쓰려면 CLI가 있으면 편합니다.

### 로컬과 원격 중 뭘 고를까요?
처음이면 **로컬**이 더 쉽습니다.
이미 서버가 있고 계속 켜둘 계획이 있다면 **원격**도 좋습니다.

### 권한을 다 허용해야 하나요?
쓰려는 기능에 따라 다르지만, 자동화·손쉬운 사용·화면 기록은 실제 도움 기능에서 자주 필요합니다.
