# 쇼케이스 (Showcase)

이 문서는 “다른 사람들은 OpenClaw를 어디까지 써봤을까?”를 보여주는 모음집입니다.

설명만 보면 감이 잘 안 올 수 있는데,
실제 사례를 보면 OpenClaw가 **단순한 챗봇이 아니라 일하는 도구**라는 점이 더 잘 보입니다.

:::tip 쉬운 비유
쇼케이스는 설명서보다 **실사용 사진첩**에 가까워요.
제품 소개보다 “사람들이 실제로 이렇게 썼다”를 보는 문서라고 생각하면 쉽습니다.
:::

## 먼저 알아두기

아래 사례 중 일부는 커뮤니티 공유를 바탕으로 정리된 내용입니다.
문서에 **(출처 미확인)** 이라고 적힌 항목은 공식 검증 전 사례입니다.

OpenClaw 소개나 설치 방법은 여기서 길게 반복하지 않겠습니다.
자세한 내용은 [설치 개요](/install/)를 참고하세요.

## 1) GitHub PR 리뷰를 Telegram으로 받기 (출처 미확인)

이 사례에서는 OpenClaw가 코드 변경 내용을 읽고, 개선할 점을 Telegram으로 보내줍니다.

### 어떤 점이 인상적일까요?
- 개발자가 GitHub 화면만 붙잡고 있지 않아도 됩니다.
- 이동 중에도 PR 요약과 피드백을 볼 수 있습니다.
- 코드 리뷰 속도를 높이는 데 도움이 됩니다.

### 실제 느낌
“PR 열어봐”가 아니라
“핵심만 메시지로 먼저 알려줘”에 가까운 흐름입니다.

## 2) 와인 셀러 관리 스킬 만들기 (출처 미확인)

CSV 데이터를 읽어 개인 와인 보관 목록을 관리하는 커스텀 스킬 사례입니다.

### 왜 재미있을까요?
이 사례는 OpenClaw가 단순 범용 봇이 아니라,
**내 취미와 생활 방식에 맞는 전용 도구**가 될 수 있다는 걸 보여줍니다.

## 3) 장보기 자동화 (출처 미확인)

브라우저 자동화를 이용해 식단 계획에서 장보기 주문, 배송 예약까지 이어진 사례입니다.

### 어떤 점이 강할까요?
- 반복 클릭을 줄일 수 있습니다.
- “매주 하는 귀찮은 일”을 줄이는 데 잘 맞습니다.
- 계획 → 주문 → 완료까지 한 흐름으로 묶을 수 있습니다.

## 4) 스크린샷을 마크다운으로 바꾸기 (출처 미확인)

화면 일부를 캡처하면 텍스트를 읽어서 마크다운 문서처럼 바꾸는 워크플로우입니다.

### 어디에 쓸 수 있을까요?
- 강의 슬라이드 정리
- 문서 초안 만들기
- 화면 속 텍스트를 빠르게 기록하기

## 5) Telegram으로 iOS 앱 빌드하기 (출처 미확인)

메신저 대화만으로 앱 빌드와 배포 흐름을 이어간 사례입니다.

### 왜 놀라운가요?
보통은 개발 도구 화면 안에서만 할 것 같은 일을,
채팅 중심 흐름으로 옮겼다는 점이 포인트입니다.

즉, OpenClaw는 “채팅창에서 말만 하는 도구”가 아니라
**작업실과 연결된 조정석**이 될 수 있습니다.

## 6) 여러 메신저를 함께 다루는 도구 만들기 (출처 미확인)

iMessage, WhatsApp 등 다양한 메신저를 하나의 흐름으로 엮는 사례도 공유되었습니다.

이런 사례는 OpenClaw가 여러 채널을 연결하는 데 강하다는 점을 보여줍니다.

## 7) 스마트 홈 제어 (출처 미확인)

Home Assistant와 비슷한 흐름으로, 자연어로 집 안 장치를 다루는 사례도 있습니다.

예를 들면:
- “거실 불 꺼줘.”
- “현재 상태 알려줘.”
- “온도 이상하면 메시지 보내줘.”

## 이 사례들이 말해주는 공통점

서로 분야는 달라도 공통점이 있습니다.

- 메신저를 일하는 창으로 바꾼다
- 파일, 브라우저, 자동화를 연결한다
- 나만의 업무 흐름에 맞게 확장할 수 있다

즉, OpenClaw는 **정해진 기능만 쓰는 도구**보다
**내 흐름에 맞게 자라나는 도구**에 가깝습니다.

:::tip ✅ 성공하면 이렇게 보입니다
쇼케이스를 보고 나서 가장 좋은 반응은 이것입니다.

“와, 대단하다”에서 끝나는 게 아니라,
“내 일에도 이걸 이렇게 바꿔 쓸 수 있겠다”라는 생각이 드는 것.

그 감이 잡히면 제대로 본 거예요.
:::
