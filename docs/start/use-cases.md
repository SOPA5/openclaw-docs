# 활용 사례 (Use Cases) 🦞

이 문서에서는 OpenClaw를 처음 시작할 때 필요한 핵심 흐름을 배웁니다.

## 📌 이 문서에서 배우는 것
- 1. 📂 개인 파일 어시스턴트 (Remote File Access)
- 2. 🔍 실시간 리서치 및 웹 요약 (Web Browsing)
- 3. 🤖 워크플로우 자동화 (Automation)

걱정하지 마세요, 하나씩 따라하면 됩니다! 😊


OpenClaw는 단순한 챗봇이 아닙니다. 자기 호스팅 Gateway를 중심으로 20개 이상의 채팅 채널, 30개 이상의 AI 모델 제공자, 그리고 로컬 리소스를 연결하여 실질적인 생산성 도구로 활용할 수 있습니다.

---


:::tip 💡 쉽게 이해하기
**Gateway**는 쉽게 말해 "교환대"예요. 여러 앱과 도구, AI 모델 사이를 이어주는 중앙 통로라고 생각하면 이해하기 쉽습니다.
:::

## 1. 📂 개인 파일 어시스턴트 (Remote File Access)

집에 있는 메인 컴퓨터에 OpenClaw를 설치해두면, 외출 중에도 Telegram이나 Slack을 통해 파일을 찾고 요약하거나 변환을 요청할 수 있습니다.

- **예시**: "지난주에 작성한 '기획서.docx' 파일 찾아서 PDF로 변환해줘."
- **이점**: 클라우드에 파일을 올리지 않아도 보안이 유지된 상태로 내 컴퓨터에 접근할 수 있습니다.

---

## 2. 🔍 실시간 리서치 및 웹 요약 (Web Browsing)

OpenClaw의 브라우저 도구와 웹 검색 기능(Brave, Firecrawl, Perplexity 등)을 활용하여 실시간 정보를 검색하고 정리합니다.

- **예시**: "오늘 테슬라 주가랑 관련 주요 뉴스 3줄 요약해서 알려줘."
- **이점**: 광고 많은 웹사이트를 직접 방문하지 않고도 메시지 창 안에서 핵심 정보만 받아볼 수 있습니다.

---

## 3. 🤖 워크플로우 자동화 (Automation)

n8n, Cron 스케줄, exec 도구를 결합하면 단순 질답을 넘어 실제로 '행동'하는 에이전트가 됩니다.

- **예시**: "나 방금 회의 끝났어. 녹음본 요약해서 팀 Slack 채널에 공유하고 할 일 목록에 추가해줘."
- **이점**: 반복적인 수작업을 메시지 한 줄로 자동화할 수 있습니다.

---

## 4. 🔗 멀티 채널 고객 응대 (Multi-Channel (메시징 채널, 즉 채팅 앱) Support)

동일한 AI 로직을 Telegram, WhatsApp, Discord, Slack 등 여러 채널에 동시 배포할 수 있습니다. OpenClaw는 단일 Gateway로 20개 이상의 채널을 지원합니다.

- **예시**: "쇼핑몰 고객이 문의하면 재고 API 확인해서 자동으로 답변해줘."
- **이점**: 채널별로 봇을 각각 개발할 필요 없이 OpenClaw 하나로 통합 관리가 가능합니다.

---

## 5. 🏠 로컬 스마트 홈 제어 (Self-Hosting)

OpenClaw는 로컬에서 실행되므로, 외부 클라우드 없이 집 안의 장치들을 제어하는 보안 비서로 활용할 수 있습니다.

- **예시**: "거실 조명 끄고 현재 서버 온도 체크해줘."
- **이점**: 프라이버시가 중요한 스마트 홈 데이터를 외부 서버로 보내지 않고 안전하게 처리합니다.

---

## 6. 📱 모바일 노드 활용 (Mobile Node)

iOS/Android 기기를 OpenClaw node로 페어링하면 Canvas, 음성 명령(Voice), 카메라 등 기기 기능을 에이전트에서 직접 활용할 수 있습니다.

- **예시**: "지금 카메라로 찍은 영수증 내용 인식해서 가계부에 추가해줘."
- **이점**: 스마트폰이 AI 에이전트의 눈과 귀가 됩니다.
