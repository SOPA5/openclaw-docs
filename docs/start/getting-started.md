# 시작하기 (Getting Started)

:::tip 설치가 끝났다면 여기서 시작하세요
**이 문서는 "설치했고, 이제 첫 설정과 첫 질문까지 가고 싶다"는 사람을 위한 가장 짧은 안내서입니다.**

- **아직 설치 전이다** → [설치 개요](/install/)부터 먼저 보세요.
- **설치는 끝났다** → 아래 **3단계 빠른 시작**부터 바로 진행하세요.
- **내 운영체제별 설치가 더 필요하다** → [플랫폼 가이드](/platforms/)에서 내 환경 문서를 먼저 확인하세요.
:::

## 한눈에 보는 3단계 빠른 시작

| 단계 | 지금 할 일 | 이동 |
|---|---|---|
| 1 | 설치가 끝났는지 확인 | [설치 개요](/install/) |
| 2 | `openclaw onboard --install-daemon` 실행 | [설정 마법사](/start/wizard) |
| 3 | 채널 연결 후 첫 질문 보내기 | [채널 페어링](/start/pairing) |

이 문서는 **설치 직후부터 첫 질문을 보내기 전까지** 막히지 않게 이어주는 출발선입니다.
복잡한 개념 설명보다, **지금 당장 무엇을 하면 되는지**에 집중합니다.

## 🎒 시작 전에 이것만 준비하면 돼요

온보딩을 시작하기 전에 아래 준비물이 있으면 훨씬 빠르게 끝납니다.

| 준비물 | 왜 필요한가요? |
|---|---|
| OpenAI / Anthropic / Google 계정 또는 API 키 | AI 모델 연결에 필요할 수 있어요 |
| Telegram 봇 토큰 | Telegram으로 대화하려면 필요해요 |
| Discord 봇 설정 정보 | Discord를 붙일 때 필요해요 |
| 테스트용 채팅방 또는 내 DM | 연결 확인을 바로 해볼 수 있어요 |

:::tip 꼭 다 준비해야 하나요?
아니요. 하지만 **"나는 Telegram으로 쓸 거야"**, **"나는 OpenAI 모델을 쓸 거야"**가 정해졌다면,
그 계정이나 토큰을 먼저 준비해두면 중간에 덜 멈춥니다.
:::

## 🚀 1단계. 온보딩 실행

설치가 끝났다면 먼저 아래 명령어를 실행하세요.

```bash
openclaw onboard --install-daemon
```

이 마법사가 보통 아래를 차례대로 도와줍니다.

- **AI 모델 연결**
- **채널 연결**
- **게이트웨이 설정**
- **백그라운드 서비스 등록**

> 더 자세한 설명이 필요하면 [설정 마법사](/start/wizard)를 함께 보세요.

---

## 🔍 2단계. 상태 확인

온보딩이 끝났다면 상태를 확인합니다.

```bash
openclaw gateway status
```

`running`이 보이면 기본 준비는 잘 된 것입니다.

원하면 대시보드도 열어볼 수 있습니다.

```bash
openclaw dashboard
```

브라우저에서 `http://127.0.0.1:18789/`가 열리면 정상입니다.

---

## 💬 3단계. 채널 연결하고 첫 질문 보내기

이제 실제로 대화할 채널을 연결하면 됩니다.

가장 많이 쓰는 예시는 이런 것들입니다.

- Telegram
- Discord
- Slack
- WhatsApp

처음이라면 아래 순서로 하면 쉽습니다.

1. [채널 페어링](/start/pairing) 문서 열기
2. 내가 쓸 채널 하나만 먼저 연결하기
3. 내 DM이나 테스트 채팅방에서 질문 보내보기

예시 질문:

- `오늘 해야 할 일 정리해줘`
- `이 링크 요약해줘`
- `내 메모를 보기 쉽게 정리해줘`

---

## ✨ 성공하면 이렇게 보입니다

성공적으로 끝나면 보통 이런 느낌입니다.

- `openclaw gateway status`에서 **running**이 보임
- 대시보드가 열림
- Telegram이나 Discord 같은 채널에서 OpenClaw가 답장함

즉, **"설치만 된 상태"**에서 **"이제 실제로 쓸 수 있는 상태"**로 넘어온 것입니다.
