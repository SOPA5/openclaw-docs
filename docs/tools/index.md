# 도구 및 기술 (Tools)

도구는 OpenClaw가 **말만 하는 AI**가 아니라 **실제로 행동하는 AI**가 되게 해주는 손과 발입니다.

<ProgressPill level="Core" progress={72} label="핵심 개념 · 실제 작업 도구 이해" />

<InfoCard title="이 문서의 역할" icon="🛠️" tone="amber">
개별 명령을 외우기보다, 어떤 종류의 도구가 있고 언제 쓰는지 한눈에 잡게 도와주는 문서입니다.
</InfoCard>

## 📌 이 문서에서 배우는 것
- 도구를 큰 분류로 보는 법
- 파일·실행·브라우저·세션 도구 차이
- 왜 권한과 안전이 중요한지
- 다음에 무엇을 읽으면 좋은지

:::tip 쉽게 이해하기
도구는 **공구 상자**와 같습니다. 망치, 드라이버, 렌치가 전부 다르듯이, `read`, `exec`, `browser`, `sessions_spawn`도 각각 쓰임새가 다릅니다.
:::

## 핵심 분류

| 분류 | 대표 도구 | 언제 쓰나 |
|---|---|---|
| 파일 | `read`, `write`, `edit`, `apply_patch` | 파일을 읽고 고칠 때 |
| 실행 | `exec`, `process` | 명령을 실행하고 긴 작업을 관리할 때 |
| 웹/브라우저 | `browser`, `web_search`, `web_fetch` | 검색, 페이지 읽기, 실제 클릭이 필요할 때 |
| 커뮤니케이션 | `message`, `tts` | 채널로 결과를 보내거나 음성으로 낼 때 |
| 세션/오케스트레이션 | `sessions_spawn`, `sessions_history`, `sessions_yield` | 다른 작업자를 띄우고 흐름을 관리할 때 |

## 왜 분류가 중요할까
- 같은 문제도 어떤 도구로 접근하느냐에 따라 속도와 안정성이 달라집니다.
- 브라우저 클릭이 필요한데 `web_fetch`만 쓰면 해결이 안 됩니다.
- 작은 수정인데 무거운 실행 흐름을 쓰면 오히려 느려집니다.

## 실전 예시
- **문서 한 줄 수정**: `read` → `edit`
- **서버 로그 확인**: `exec` → 필요하면 `process`
- **로그인된 사이트 조작**: `browser`
- **큰 작업 나누기**: `sessions_spawn`

## ✅ 성공 확인
- 어떤 문제를 봤을 때 먼저 어떤 도구 묶음을 써야 할지 감이 옵니다.
- 읽기 중심인지, 실행 중심인지, 브라우저 상호작용 중심인지 구분할 수 있습니다.
- 세션 도구가 일반 실행 도구와 왜 다른지 설명할 수 있습니다.
