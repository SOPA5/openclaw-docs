# 서브 에이전트 (Sub-agents)

서브에이전트는 큰 일을 **작은 일로 나눠 맡기는 작업자**입니다.

## 📌 이 문서에서 배우는 것
- 언제 서브에이전트를 쓰는지
- `sessions_spawn`이 왜 중요한지
- 병렬 작업에서 무엇을 조심해야 하는지

:::tip 쉽게 이해하기
서브에이전트는 **팀 프로젝트 조별 역할 나누기**와 같습니다. 한 사람이 다 하면 느리지만, 조사 담당·검토 담당으로 나누면 더 빨라질 수 있습니다.
:::

## 핵심 이해
- 크고 독립적인 작업 조각이 있을 때 가장 효과적입니다.
- `sessions_spawn`으로 새 작업자를 띄우고 결과를 받습니다.
- 서로 강하게 얽힌 작업은 무작정 나누면 오히려 느려질 수 있습니다.

## 실전 예시
- **작은 테스트 먼저**: 운영 전에 가장 작은 범위에서 먼저 확인합니다.
- **도구 선택 점검**: 정말 이 도구가 맞는지, 더 단순한 도구로 해결되는지 먼저 봅니다.


## ✅ 성공 확인
- 이 도구를 언제 써야 하는지와 언제 다른 도구를 써야 하는지 구분할 수 있습니다.
- 기본 흐름이나 준비물이 머릿속에 그려집니다.
- 작은 테스트로 먼저 검증해야 한다는 원칙을 이해했습니다.
