# Canvas와 A2UI

이 문서에서는 OpenClaw가 **작은 화면이나 UI를 보여주는 방식**을 배웁니다.

## 📌 이 문서에서 배우는 것
- Canvas가 무엇인지
- A2UI가 무슨 뜻인지
- Canvas를 언제 쓰는지
- 실전에서 어떤 화면을 만들 수 있는지

:::tip 💡 쉽게 이해하기
- **Canvas** = 에이전트가 만든 화면을 보여주는 칠판
- **A2UI** = Agent to UI, 즉 **에이전트가 UI로 정보를 보내는 방식**

쉽게 말해, AI가 말만 하는 게 아니라 **직접 작은 화면도 보여 주는 것**입니다.
:::

## Canvas란?

Canvas는 에이전트가 만든 결과를 **눈으로 보기 쉽게 보여주는 화면 공간**입니다.

그냥 긴 글로 설명하는 대신,
예를 들면 이런 것을 보여줄 수 있습니다.

- 진행 상태 보드
- 요약 카드
- 버튼이 있는 간단한 화면
- 작은 대시보드
- 시각적인 결과판

즉, Canvas는 **“텍스트 답변”을 “보이는 화면”으로 바꿔 주는 자리**라고 생각하면 됩니다.

## 왜 좋은가요?

어떤 정보는 글보다 화면이 훨씬 이해가 빠릅니다.

예를 들어:
- 10줄 설명보다 카드 3개가 빠를 때
- 단계별 진행 상황을 표처럼 보여 주고 싶을 때
- 사용자가 버튼을 눌러 다음 행동을 고르게 하고 싶을 때

그래서 Canvas는 **설명용**, **데모용**, **상태 표시용**으로 특히 좋습니다.

## A2UI란?

A2UI는 **Agent to UI**의 줄임말입니다.
이 말은, 에이전트가 자기 결과를 **화면 구조로 밀어 넣는 방식**을 뜻합니다.

쉽게 말하면:

- 그냥 말하기만 하는 AI → 텍스트만 반환
- A2UI를 쓰는 AI → 화면에 보여줄 데이터도 함께 보냄

그래서 사용자는 단순 답변뿐 아니라,
**눈으로 바로 이해할 수 있는 UI 결과**를 보게 됩니다.

## Canvas는 언제 쓰면 좋을까요?

아래 같은 상황에서 잘 맞습니다.

### 1. 상태를 보여주고 싶을 때
예:
- 작업 진행률
- 서버 상태
- 체크리스트 완료 여부

### 2. 요약을 카드로 보여주고 싶을 때
예:
- 오늘 할 일 3개
- 핵심 메모 5개
- 분석 결과 요약

### 3. 사용자가 선택하게 하고 싶을 때
예:
- 버튼 선택
- 간단한 액션 메뉴
- 다음 단계 안내

## Canvas 사용 흐름

보통 생각은 아주 단순합니다.

1. 에이전트가 결과를 만듭니다.
2. 그 결과를 Canvas에 맞는 화면 형태로 보냅니다.
3. 사용자는 화면으로 봅니다.
4. 필요하면 버튼이나 다음 액션으로 이어집니다.

즉, **생각은 에이전트가 하고, 보여주기는 Canvas가 맡는 구조**예요.

:::tip 처음에는 작게 시작하세요
처음부터 복잡한 대시보드를 만들기보다,
**카드 1개**, **상태 박스 1개**, **버튼 2개** 정도로 시작하면 훨씬 쉽습니다.
:::

## 실전 예시

### 예시 1. 작업 진행 보드

에이전트가 아래를 화면으로 보여줄 수 있습니다.

- 해야 할 일
- 진행 중
- 완료됨

글로 길게 말하는 대신,
사용자는 한눈에 상태를 봅니다.

### 예시 2. 회의 요약 카드

회의가 끝난 뒤:
- 핵심 결정 3개
- 담당자
- 마감일

이런 내용을 카드 형태로 보여주면 훨씬 빨리 이해됩니다.

### 예시 3. 간단한 컨트롤 패널

예를 들어:
- 새로고침 버튼
- 상태 보기 버튼
- 다음 단계 버튼

이런 식으로 아주 작은 UI도 만들 수 있습니다.

## Canvas가 특히 잘 맞는 장면

- 데모를 보여줄 때
- 사용자가 긴 글을 싫어할 때
- 상태판이 계속 바뀔 때
- 에이전트 결과를 더 직관적으로 보여주고 싶을 때

## 주의할 점

- 너무 많은 정보를 한 화면에 넣지 않기
- 텍스트만으로 충분한 작업은 굳이 복잡하게 만들지 않기
- 처음에는 작고 분명한 UI부터 만들기

## ✅ 성공 확인

아래처럼 되면 잘 이해한 것입니다.

- Canvas가 “에이전트 결과를 보여주는 화면”이라는 점을 이해합니다.
- A2UI가 “에이전트가 UI로 결과를 보내는 방식”이라는 점을 이해합니다.
- 글보다 화면이 더 좋은 상황을 구분할 수 있습니다.
- 작은 카드/상태판/버튼 UI부터 시작해야 한다는 원칙을 알게 됩니다.
