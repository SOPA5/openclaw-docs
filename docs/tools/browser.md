# 브라우저 도구 (Browser Tool)

브라우저 도구는 OpenClaw가 **웹페이지를 실제로 보고, 누르고, 입력하는** 데 쓰는 도구입니다.

## 📌 이 문서에서 배우는 것
- 언제 `browser`를 쓰는지
- 격리 브라우저와 사용자 브라우저 차이
- 왜 `snapshot → act` 흐름이 중요한지

:::tip 쉽게 이해하기
브라우저 도구는 사람이 웹사이트를 직접 만지는 **가상 손가락**이라고 생각하면 쉽습니다. 단, 아무 데나 막 누르는 게 아니라 화면 구조를 보고 움직입니다.
:::

## 핵심 이해
- 로그인이나 클릭이 필요하면 `browser`가 맞습니다.
- 로그인 세션이 필요 없으면 기본 격리 브라우저가 더 안전합니다.
- 먼저 화면 구조를 읽고(`snapshot`), 그다음 행동(`act`)하는 흐름이 안정적입니다.

## 실전 예시
- **작은 테스트 먼저**: 운영 전에 가장 작은 범위에서 먼저 확인합니다.
- **도구 선택 점검**: 정말 이 도구가 맞는지, 더 단순한 도구로 해결되는지 먼저 봅니다.


## ✅ 성공 확인
- 이 도구를 언제 써야 하는지와 언제 다른 도구를 써야 하는지 구분할 수 있습니다.
- 기본 흐름이나 준비물이 머릿속에 그려집니다.
- 작은 테스트로 먼저 검증해야 한다는 원칙을 이해했습니다.
