# 브라우저 도구 (Browser Tool)

이 문서에서는 OpenClaw 도구를 안전하고 효과적으로 쓰는 방법을 배웁니다.

## 📌 이 문서에서 배우는 것
- 두 가지 주요 프로필
- 기본 프로필: OpenClaw 관리 브라우저
- `profile="user"`

걱정하지 마세요, 하나씩 따라하면 됩니다! 😊


OpenClaw의 `browser` 도구는 단순 스크린샷 유틸이 아닙니다. **OpenClaw가 관리하는 브라우저 인스턴스와 실제 사용자 브라우저 프로필을 모두 다룰 수 있는 웹 자동화 표면**입니다.

## 두 가지 주요 프로필

### 기본 프로필: OpenClaw 관리 브라우저

프로필을 지정하지 않으면 보통 OpenClaw가 관리하는 격리 브라우저(`openclaw`)를 사용합니다. 자동화 재현성, 안전성, 독립성이 좋습니다.

### `profile="user"`

이미 로그인된 실제 사용자 브라우저 세션이 필요할 때 사용합니다. 예를 들어 사용자의 쿠키, 로그인 상태, 열린 탭이 중요할 때 유용합니다.

다만 이 모드는 민감도가 높으므로, 자동화 전에 **정말 기존 로그인 세션이 필요한지** 판단하는 것이 좋습니다.

## 핵심 동작 모델: snapshot + act

최신 사용 패턴은 브라우저를 “좌표 클릭”보다 **구조화된 스냅샷 기반 UI 제어**로 다루는 것입니다.

### `snapshot`

현재 페이지의 접근성 트리나 구조화된 UI 상태를 읽습니다. 어떤 요소가 보이고, 어떤 레퍼런스로 조작 가능한지 파악할 때 씁니다.

### `act`

스냅샷에서 얻은 ref를 바탕으로 클릭, 입력, 선택, 키 입력 같은 동작을 수행합니다.

이 패턴의 장점:

- 페이지 구조를 보고 행동할 수 있음
- 좌표 기반 자동화보다 안정적임
- 같은 탭에서 ref를 유지하며 작업하기 쉬움

## 주요 용도

- 로그인된 사이트 탐색
- Control UI 조작
- 폼 입력 자동화
- 스크린샷/증거 수집
- 탭 전환 및 특정 브라우저 상태 확인

## 브라우저 도구와 일반 웹 도구의 차이

- `web_search` / `web_fetch`는 **본문 수집과 검색**에 강함
- `browser`는 **실제 인터랙션**에 강함

즉, 클릭·입력·탭 관리가 필요하면 `browser`, 읽기 중심이면 `web_fetch`나 `web_search`가 더 적합합니다.

## 실무 팁

- 로그인/쿠키가 중요하지 않으면 기본 격리 프로필 우선
- 로그인 세션이 꼭 필요할 때만 `profile="user"`
- 자동화는 `snapshot` → `act` 순서로 설계
- 같은 탭에서 얻은 ref는 가능한 한 같은 target에서 계속 사용
