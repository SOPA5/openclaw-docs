# 도구 샌드박스 (Sandbox)

샌드박스는 도구가 **어디까지 접근할 수 있는지 경계선을 그어 주는 안전장치**입니다.

## 📌 이 문서에서 배우는 것
- 샌드박스가 왜 필요한지
- 권한과 정책을 어떻게 이해해야 하는지
- 실행 도구와 함께 볼 때 주의할 점

:::tip 쉽게 이해하기
샌드박스는 **놀이터 울타리**와 비슷합니다. 아이가 마음껏 뛰어놀 수 있지만, 위험한 바깥으로는 함부로 나가지 못하게 해 줍니다.
:::

## 핵심 이해
- 도구가 강력할수록 경계도 분명해야 안전합니다.
- 허용된 범위 안에서만 파일·명령·브라우저 접근이 이루어집니다.
- 안전 정책을 이해하면 승인 흐름도 덜 헷갈립니다.

## 실전 예시
- **작은 테스트 먼저**: 운영 전에 가장 작은 범위에서 먼저 확인합니다.
- **도구 선택 점검**: 정말 이 도구가 맞는지, 더 단순한 도구로 해결되는지 먼저 봅니다.


## ✅ 성공 확인
- 이 도구를 언제 써야 하는지와 언제 다른 도구를 써야 하는지 구분할 수 있습니다.
- 기본 흐름이나 준비물이 머릿속에 그려집니다.
- 작은 테스트로 먼저 검증해야 한다는 원칙을 이해했습니다.
