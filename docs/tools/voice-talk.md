# Voice Wake와 Talk Mode

이 문서에서는 OpenClaw와 **말로 대화하는 방법**을 배웁니다.

## 📌 이 문서에서 배우는 것
- Talk Mode가 무엇인지
- Voice Wake가 어떤 역할을 하는지
- TTS와 STT가 무엇인지
- 어떤 제공자를 붙일 수 있는지
- 실전에서 어떻게 활용하는지

:::tip 💡 쉽게 이해하기
- **Talk Mode** = AI와 말로 대화하는 모드
- **STT** = 내가 한 말을 글자로 바꾸는 기술
- **TTS** = AI가 할 말을 소리로 읽어 주는 기술

즉, **내 입 → 글자 → AI → 글자 → 목소리** 흐름이라고 보면 됩니다.
:::

## Talk Mode란?

Talk Mode는 쉽게 말해 **키보드 대신 목소리로 AI와 대화하는 방식**입니다.

보통 흐름은 이렇게 됩니다.

1. 내가 말함
2. STT가 말을 글자로 바꿈
3. OpenClaw가 답변 생성
4. TTS가 답변을 소리로 읽어 줌

그래서 손이 바쁠 때, 이동 중일 때, 화면을 오래 보기 싫을 때 특히 편합니다.

### 언제 유용할까요?

- 운전 전후 메모 정리
- 산책하면서 아이디어 말하기
- 요리하면서 질문하기
- 책상에서 핸즈프리 작업하기
- 모바일에서 더 자연스럽게 대화하기

## Voice Wake란?

Voice Wake는 **“이제부터 내 말을 들어 줘”**라고 AI를 깨우는 장치입니다.

쉽게 말하면 스마트 스피커의 호출어와 비슷한 개념이에요.
예를 들면:
- 특정 단어를 말하면 듣기 시작
- 버튼을 누르면 말하기 시작
- 짧은 트리거 후 음성 입력 시작

### 왜 필요한가요?

항상 마이크를 열어 두면 편할 수도 있지만,
원하지 않는 소리까지 들어갈 수 있습니다.
그래서 Voice Wake는 **언제부터 듣기 시작할지** 정하는 역할을 합니다.

### Voice Wake 설정에서 생각할 것

- 호출어를 쓸지
- 버튼형으로 쓸지
- 자동 시작을 쓸지
- 주변 소음이 많은 환경인지

:::tip 처음에는 단순하게
처음 세팅할 때는 **항상 켜진 마이크**보다
**버튼으로 시작하는 방식**이 더 덜 헷갈리고 실수도 적습니다.
:::

## TTS 설정: 텍스트를 음성으로 읽기

TTS는 **Text To Speech**의 줄임말입니다.
즉, 글자를 목소리로 바꾸는 기술이에요.

### TTS가 하는 일

- AI 답변을 소리로 읽어 줌
- 채팅 내용을 음성으로 바꿈
- 손이나 눈을 덜 쓰고도 응답을 들을 수 있게 해 줌

### TTS를 설정할 때 보는 것

- 목소리 품질
- 응답 속도
- 한국어/영어 지원
- 비용
- 감정 표현이나 자연스러움

### 대표 제공자 예시

- **ElevenLabs**: 목소리가 자연스럽고 표현력이 좋음
- **OpenAI 계열 음성 기능**: 통합이 쉬운 편
- 기타 TTS 제공자: 환경에 따라 선택 가능

### 언제 좋은 선택일까요?

- 사람이 말하는 것처럼 자연스러운 음성이 중요하다 → ElevenLabs 계열 고려
- 이미 한 제공자에 많이 묶어 운영 중이다 → 같은 생태계로 맞추기

## STT 설정: 음성을 텍스트로 바꾸기

STT는 **Speech To Text**의 줄임말입니다.
즉, 내가 한 말을 글자로 바꾸는 기술입니다.

### STT가 하는 일

- 마이크로 받은 말을 글자로 변환
- Talk Mode의 입력 부분 담당
- 음성 메모나 명령을 텍스트 질문으로 바꿔 줌

### STT를 설정할 때 보는 것

- 인식 정확도
- 한국어 지원
- 주변 소음에 얼마나 강한지
- 처리 속도
- 긴 말도 안정적으로 받는지

### 대표 제공자 예시

- **OpenAI Whisper**: 널리 알려진 음성 인식 계열
- 기타 STT 제공자: 실시간성, 비용, 언어 지원에 따라 선택

:::tip 좋은 시작점
처음에는 **정확도가 좋은 STT 하나 + 자연스러운 TTS 하나**로 조합해 보는 것이 가장 쉽습니다.
예를 들면 **Whisper + ElevenLabs** 같은 식이에요.
:::

## 지원 제공자 예시

환경에 따라 다르지만, 보통 아래 같은 이름을 많이 보게 됩니다.

### TTS 쪽
- ElevenLabs
- OpenAI 음성 계열
- 기타 외부 TTS 제공자

### STT 쪽
- OpenAI Whisper
- 기타 실시간/배치 STT 제공자

중요한 점은 **“무조건 최고 하나”**가 있는 게 아니라,
아래 3가지를 보고 고르는 것입니다.

1. 정확도
2. 자연스러움
3. 비용과 속도

## 실전 활용 예시

### 예시 1. 걷다가 아이디어 저장

- Voice Wake 또는 버튼으로 시작
- “지금 떠오른 랜딩페이지 문구 정리해 줘”라고 말함
- STT가 텍스트로 바꿈
- OpenClaw가 정리
- TTS가 핵심만 읽어 줌

### 예시 2. 책상에서 핸즈프리 작업

- 손은 다른 작업 중
- “오늘 할 일 3개만 정리해 줘”라고 말함
- 답변을 음성으로 들음

### 예시 3. 이동 중 빠른 질의응답

- 휴대폰에서 짧게 질문
- 긴 타이핑 없이 음성으로 대화
- 답변도 귀로 바로 확인

## 잘 쓸 때의 팁

- 짧고 분명하게 말하기
- 시끄러운 곳에서는 이어폰 마이크 쓰기
- 처음에는 명령형보다 평범한 문장으로 말하기
- 너무 긴 문장보다 한 번에 한 가지 요청하기

## 주의할 점

- 주변 사람 목소리가 같이 들어갈 수 있음
- 민감한 정보는 공공장소에서 말하지 않는 것이 좋음
- TTS/STT 제공자별 비용이 다를 수 있음
- 실시간성은 네트워크 상태 영향을 받을 수 있음

## ✅ 성공 확인

아래처럼 되면 잘 설정된 것입니다.

- 말한 내용이 텍스트로 잘 바뀝니다.
- OpenClaw 답변이 음성으로 자연스럽게 들립니다.
- Voice Wake 또는 시작 방식이 헷갈리지 않습니다.
- 이동 중이나 핸즈프리 상황에서 실제로 편합니다.
