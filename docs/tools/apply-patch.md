# apply_patch 도구

이 문서에서는 OpenClaw 도구를 안전하고 효과적으로 쓰는 방법을 배웁니다.

## 📌 이 문서에서 배우는 것
- 왜 중요한가
- 멀티 헝크 패치란?
- 언제 특히 유리한가

걱정하지 마세요, 하나씩 따라하면 됩니다! 😊


`apply_patch`는 OpenClaw의 정밀 편집 도구입니다. 특히 **여러 파일의 여러 위치를 한 번에 바꾸는 멀티 헝크 패치(multi-hunk patch)** 작업에 강합니다.

## 왜 중요한가

긴 파일을 수정할 때 전체 파일을 매번 다시 쓰면 다음 문제가 생깁니다.

- 토큰 비용 증가
- 의도하지 않은 줄 손상
- 큰 파일에서 정확도 저하
- 여러 위치 동시 수정의 불편함

`apply_patch`는 필요한 조각만 바꾸기 때문에 더 안전하고 효율적입니다.

## 멀티 헝크 패치란?

하나의 패치 안에 **여러 수정 블록(hunk)** 을 담는 방식입니다. 예를 들어 다음처럼 쓸 수 있습니다.

- 파일 A의 import 수정
- 같은 파일 A의 함수 본문 수정
- 파일 B에 새 설정 추가

이 모든 변경을 한 번의 구조화된 패치로 적용할 수 있습니다.

## 언제 특히 유리한가

- 대형 소스 파일의 국소 수정
- 여러 문서의 반복 표현 교체
- 리팩터링 중 소규모 변경 다발 처리
- 정확한 before/after 문맥이 중요할 때

## `edit` / `write`와의 차이

- `write`: 파일 전체를 새로 쓸 때 적합
- `edit`: 정확한 문자열 치환 한두 곳에 적합
- `apply_patch`: 여러 군데를 구조적으로 바꿀 때 적합

즉, 세 도구는 경쟁 관계가 아니라 **편집 범위와 복잡도에 따라 나뉘는 선택지**입니다.

## 좋은 사용 기준

- 파일 전체를 갈아엎는 개정이면 `write`
- 한 문장/한 블록 정밀 치환이면 `edit`
- 여러 위치를 한 번에 손보면 `apply_patch`
