# BlueBubbles

BlueBubbles는 OpenClaw를 **실제 대화창에 들여보내는 방법** 중 하나입니다. 어떤 채널이든 원리는 비슷하지만, BlueBubbles는 특히 **Apple 메시지 경험을 살리고 싶을 때** 잘 맞습니다.

## 📌 이 문서에서 배우는 것
- 언제 BlueBubbles를 쓰면 좋은지
- 연결 준비물은 무엇인지
- 처음 테스트를 어떻게 하면 좋은지
- 운영할 때 주의할 점

:::tip 쉽게 이해하기
채널 연결은 **새 친구를 집에 초대하는 것**과 비슷합니다. 주소와 출입문 비밀번호가 맞아야 들어올 수 있듯이, 토큰·시크릿·앱 설정이 맞아야 OpenClaw가 대화방에 들어올 수 있습니다.
:::

## 언제 쓰면 좋은가
- Apple 메시지 경험을 살리고 싶을 때
- 신규 iMessage 연결 경로를 찾고 있을 때
- macOS 기반 브리지를 운영할 수 있을 때

## 설정 개요
1. BlueBubbles 서버를 실행할 macOS 장치를 준비합니다.
2. BlueBubbles 쪽 연결 정보와 OpenClaw 쪽 설정 값을 준비합니다.
3. OpenClaw에 채널 정보를 연결합니다.
4. Apple 메시지 테스트 대화에서 송수신을 확인합니다.

## ✅ 성공 확인
- 테스트 대화에서 메시지를 보냈을 때 응답이 돌아옵니다.
- 연결 오류 없이 채널 앱 또는 봇이 정상 상태로 보입니다.
- DM 또는 작은 테스트 방에서 기본 질의응답이 됩니다.

## 운영 팁
- 신규 구성에서는 iMessage legacy보다 BlueBubbles를 먼저 검토하세요.
- macOS 장치가 계속 켜져 있어야 운영이 안정적입니다.
- 처음에는 본인 계정끼리 테스트하는 편이 안전합니다.

## 실전 예시
- **개인 비서형**: 나만 보는 테스트 대화에서 일정 요약이나 알림을 받습니다.
- **운영형**: 작은 테스트 방에서 팀 알림, 질문 답변, 상태 확인 흐름을 먼저 검증합니다.

## 주의할 점
- 처음부터 공개 채널이나 큰 그룹에 넣지 마세요.
- 토큰·시크릿은 일반 채팅방에 올리지 마세요.
- 채널별 멘션 규칙, 답장 규칙, 알림 빈도를 먼저 정해두면 운영이 편합니다.
