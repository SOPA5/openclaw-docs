# WhatsApp

이 문서에서는 OpenClaw를 다양한 채팅 앱과 연결하는 방법을 배웁니다.

## 📌 이 문서에서 배우는 것
- 언제 쓰면 좋은가
- 설정 개요
- 운영 팁

걱정하지 마세요, 하나씩 따라하면 됩니다! 😊


WhatsApp은 실제 일상 메신저 흐름에 가장 자연스럽게 붙는 채널 중 하나입니다. 고객 응대, 개인 비서, 알림 수신 같은 사용 사례에 잘 맞습니다.


:::tip 💡 쉽게 이해하기
**Gateway**는 쉽게 말해 "교환대"예요. 여러 앱과 도구, AI 모델 사이를 이어주는 중앙 통로라고 생각하면 이해하기 쉽습니다.
:::

## 언제 쓰면 좋은가

- 개인 비서형 사용
- 글로벌 메신저 기반 알림
- 실제 사용자 메시지 흐름에 가까운 운영

## 설정 개요

1. WhatsApp 연동에 필요한 계정과 채널 앱 정보를 준비합니다.
2. OpenClaw 설정에서 WhatsApp 채널 항목에 토큰 또는 연결 정보를 입력합니다.
3. 첫 메시지는 개인 DM에서 보내 연결 상태를 확인합니다.
4. 필요하면 다른 채널과 병행해 멀티채널 운영을 구성합니다.

## 운영 팁

- 설치와 서비스 구성은 `openclaw onboard --install-daemon` 흐름을 우선 사용하세요.
- 연결 후에는 `openclaw gateway status`로 Gateway가 정상인지 먼저 확인하세요.
- 첫 테스트는 DM에서 짧은 메시지로 시작하면 문제를 가장 빨리 분리할 수 있습니다.
- 여러 채널을 동시에 붙여도 되지만, 초기 검증은 한 채널씩 순서대로 하는 편이 안전합니다.

## 주의할 점

- 채널 특성상 운영 계정과 개인 계정을 분리하는 편이 안전합니다.
- 그룹 운영 전에는 mention/reply 정책을 먼저 정하세요.
- 알림성 메시지가 많다면 과도한 발송 빈도를 피하세요.
## 🔒 보안 설정
- **DM 페어링**: 개인 메시지로 봇과 1:1 연결
- **allowFrom**: 허용된 사용자만 접근 가능하도록 설정
- **그룹 정책**: 그룹 채팅 시 mention/reply 규칙 설정
