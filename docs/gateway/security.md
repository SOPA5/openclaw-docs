# 보안 감사 및 강화 (Security)

이 문서에서는 OpenClaw Gateway가 어떻게 동작하는지 배웁니다.

## 📌 이 문서에서 배우는 것
- 가장 먼저 확인할 4가지
- DM pairing이 중요한 이유
- `allowFrom`: 네트워크 단위 안전장치

걱정하지 마세요, 하나씩 따라하면 됩니다! 😊


OpenClaw 보안의 핵심은 “모든 것을 막는다”가 아니라, **Gateway를 중심으로 접근 경로·인증·도구 권한·실행 격리를 함께 관리하는 것**입니다. 특히 멀티채널 제품이기 때문에 채널 페어링 정책과 도구 정책을 따로 보면 안 됩니다.


:::tip 💡 쉽게 이해하기
**Gateway**는 쉽게 말해 "교환대"예요. 여러 앱과 도구, AI 모델 사이를 이어주는 중앙 통로라고 생각하면 이해하기 쉽습니다.
:::

## 가장 먼저 확인할 4가지

1. **Gateway가 어디에 바인딩되어 있는가**
2. **`auth.mode`가 어떻게 설정되어 있는가**
3. **채널/노드 페어링이 얼마나 제한적인가**
4. **도구 실행이 어떤 정책과 샌드박스 아래에 있는가**

이 네 가지를 함께 봐야 실제 위험도를 판단할 수 있습니다.

## DM pairing이 중요한 이유

OpenClaw는 채널 연결 시 **DM pairing** 같은 안전한 승인 흐름을 중요하게 다룹니다. 이 방식의 장점은 다음과 같습니다.

- 누구나 임의로 봇을 붙잡고 민감한 작업을 시키기 어렵습니다.
- 개인 계정과 실제 사용자를 연결하는 승인 단계가 생깁니다.
- 그룹 채팅이나 공개 공간보다 공격면이 훨씬 작습니다.

실무적으로는 Telegram 같은 빠른 셋업 채널에서도 **DM 기반 페어링**을 기본으로 두는 것이 좋습니다.

## `allowFrom`: 네트워크 단위 안전장치

인증이 있더라도 네트워크 범위를 제한하면 훨씬 안전해집니다. `allowFrom`은 “어디서 오는 요청까지 믿을 것인가”를 정하는 층입니다.

대표적인 사용 예:

- 로컬 네트워크만 허용
- Tailscale tailnet 대역만 허용
- 리버스 프록시/게이트웨이 IP만 허용

이 설정은 인증을 대체하지 않지만, **인증 우회·오노출·잘못된 공개 URL** 사고의 피해 범위를 줄여 줍니다.

## `openclaw doctor`를 정기 감사 도구로 보기

최신 운영 흐름에서는 별도 보안 명령 하나보다 **`openclaw doctor`를 종합 점검 도구로 사용하는 해석**이 더 정확합니다.

아래 명령어를 터미널에 입력하세요:
```bash
openclaw doctor
```

doctor는 보통 다음 범주의 문제를 빠르게 잡는 데 유용합니다.

- 서비스 설치/실행 이상
- 잘못된 Gateway 노출 방식
- 인증 누락 또는 약한 구성
- 권한/환경 변수/비밀값 로딩 문제
- 채널 연결 이상

즉, doctor는 “고장 수리용”이면서 동시에 **운영 감사의 첫 번째 체크포인트**입니다.

## 채널 보안과 도구 보안은 연결되어 있습니다

예를 들어 Telegram DM 페어링이 느슨하거나, 공개 WebChat이 넓게 열려 있는데 `exec`, `browser`, `message` 같은 강한 도구가 폭넓게 허용되어 있으면 위험이 커집니다. 반대로 외부 노출이 있더라도:

- 인증이 켜져 있고
- 페어링이 엄격하며
- 에이전트별 도구 허용 범위가 좁고
- 샌드박싱이 적용되어 있으면

실제 리스크는 크게 줄어듭니다.

## 운영자가 챙겨야 할 보안 체크리스트

### 네트워크

- Gateway는 가능하면 loopback bind 유지
- 외부 공개는 Tailscale Serve/Funnel 또는 신뢰된 프록시로만 처리
- 필요 시 `allowFrom`으로 출발지 제한

### 인증

- `auth.mode` 활성화
- 비밀값은 파일/환경변수/SecretRef 계층으로 관리
- 무인증 공개 URL 금지

### 채널/노드

- DM pairing 우선
- 그룹/공용 채널은 멘션, 승인, 권한 범위를 신중하게 설정
- node pairing은 승인 흐름을 좁게 유지

### 도구

- 민감한 도구는 최소 권한 원칙 적용
- sandbox 및 tool policy 함께 사용
- elevated 실행은 정말 필요한 경우에만 허용
